Using the Pattern Library

The pattern library is baked into pages in the University CMS, so just use the markup in your content. On other pages, include the pattern library style sheet on any page with the <link> element below and the use the appropriate HTML element/class structure (as indicated in the examples). It should be the last stylesheet to load, if possible.

A word on fonts

The GVSU website uses webfonts for headings. If you are creating content for the CMS, your fonts will be fine. If you are building outside the CMS, you may need to add the webfonts and font CSS to the server you are working on. See the Typography section for more details.


<link rel="stylesheet" type="text/css" href="http://gvsu.edu/cms3/assets/741ECAAE-BD54-A816-71DAF591D1D7955C/libui.css" />

Writing readable code

  • Use classes for CSS styles. Use ids for semantic purposes and for in-page navigation.
  • Alphabetize attributes within an element. For example: <div class="line" id="lib-access" title="Accessibility Information">. The exception to this is the <a> element, which should start with the href attribute; alphabetize the attributes that follow.
  • Alphabetize multiple classes within the class attribute. For instance: <div class="accordion-body codeblock lib-button monospace">
  • It’s nice to follow XHTML syntax, but not required.

Accessibility

Use the following tips when writing HTML to improve accessibility (as a federally funded institution, we must adhere to the ADA).

  • All images must have alt attributes. Describe the image, don’t just repeat the filename.
  • For navigation items and forms, if possible, use the tabindex attribute to make it easy to navigate without a mouse.

  • Use the <abbr> element for all abbreviations and acronyms!
  • All elements in forms should have a corresponding <label> (except submit buttons).

Typography

Use standard HTML elements without any inline styles. A few notes:

  1. Don’t use <h1>.
  2. <h2> is reserved for the page title at the top of the #page-content div. Please do not repeat it.
  3. When possible, please use typographically correct punctuation:
    • Apostophe: ’ &#8217;
    • Single Quotes: ‘ ’ &#8216; &#8217;
    • Double Quotes: “ ” &#8220; &#8221;
    • Em Dash: — &#8212;
    • En Dash: – &#8211;
    • Signify file types by wrapping the file extension in a span.ftype: <span class="ftype"><abbr title="Portable Document Format">PDF</abbr></span>

Adding Webfonts to your non-CMS Page

GVSU uses the web font Alternate Gothic 3 for many of its headings. If you are building a new page or site on a non-GVSU server, you'll need to include the font stylesheet and webfonts on the server.

You should have the files for the web fonts. If not, ask Matthew for them.

Upload the fonts and the stylesheet to the same folder on the server your page will be on. Then update the path and include the following <link> element in the <head> of your page:

<link rel="stylesheet" type="text/css" href="PATH/TO/FONT/DIRECTORY/gvsu-fonts.css">

Forms

Forms are the backbone of the library website. Every system that our users have to create an account with or input data into is a potential place we will lose them. Make them simple. Never use Clear or Reset buttons. They confuse users and lead to errors.

Notes

Use placeholders with text fields if you cannot have a label, but do not supply examples as placeholders. Instead, use the label as the placeholder. Still include a label in the markup but use javascript or CSS to hide it. (Read more about this.)

Also, use columns when possible to make forms appear shorter. Make sure to use tabindexes as well.

Simple Forms

When a single text input and a single submit button are all you need for a form, use the .lib-simple-form pattern. If you need a select box, a text input, and a submit, use the .lib-book-form pattern. (We use these on our "Find Articles" and "Find Books" pages, for instance.)

Sample Markup

<form class="lib-simple-form">

Buttons

The button classes .lib-button* and .lib-button-small* can be applied to <a>, <span>, and <input type="submit">s. Buttons that make a server request (submit buttons, links to new pages) should all get blue buttons .lib-button or .lib-button-small. Buttons that trigger actions within a page (accordions etc.) should get the grey buttons (.lib-button-grey or .lib-button-small-grey). When you have two buttons, make the preferred button blue.

Sample Markup
			
	<input class="lib-button" name="post" type="submit" value="Find It" />
.lib-button-small .lib-button-small-grey
.lib-button .lib-button-grey

Use this class when you want a submit button to be the width of the viewport on small screens, but enclose it in a grid column so that it sizes properly on larger screens.

.lib-button-wide

Tables

Please don’t use tables for layout. Puppies die when you do that. If you can’t avoid using tables (for instance, for tabular data!), use inline media queries to hide non-essential columns on smaller screens, depending on your table's content.

Sample Markup

<style>
.col2 { display: none; }
@media screen and (min-width: 45em) { .col2 { display: table-cell; }}
</style>
<div class="lib-table">
	<table>	
		<thead>
			<tr>
				<th colspan="3">div.lib-table table</th>
			</tr>
			<tr class="lib-row-headings">
				<th>tr.lib-row-headings th</th>
				<th class="col2>Column 2</th>
				<th>Column 3</th>
			</tr>
		</thead>
		<tbody>
			<tr>
				<td>One</td>
				<td class="col2">One</td>

				<td>Two</td>

Status Messages

Success, warning, and error messages give feedback to the user.

Sample Markup

	<div class="lib-alert">
		<p>The library is closed due to the weather. Please check the <abbr title="Grand Valley State University">GVSU</abbr> weather hotline for updates, and we'll reopen as soon as we can dig the door out.</p>
	</div>

	<div class="lib-success">
		<p>All right! Your Document Delivery request has been placed. We'll let you know as soon as it arrives.</p>
	</div>
.lib-note Just a little information we want to highlight. This might include a link.
.lib-success Something went well, like a request was placed or the user won the lotto. This might include a link.
.lib-alert We want to tell the user something important, like the library is closed or the ASRS is down. This might include a link.
.lib-error Something has gone horribly wrong. This might include a link.
